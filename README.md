# 像素精灵表编辑器

一个面向 2D RPG 像素素材后处理的本地 Web 工具。它可以帮助你处理 AI 生成的像素人物精灵图，包括去除背景色、检查杂色、手动擦除、切分帧、重组帧、多行导出和动画预览。

## 主要功能

- 导入本地像素精灵图，并以像素清晰的方式放大预览。
- 点击画布取背景色，支持容差去背景，并将匹配像素转为透明。
- 支持背景预览开关，用指定颜色检查角色边缘或内部杂色。
- 支持去背景色边，可减轻红底、蓝底等背景扣除后残留的角色轮廓色边。
- 支持自动去毛刺，可清理孤立像素、小连通块和低 Alpha 边缘噪点。
- 支持按切分帧进行底部居中自动对齐，减少动作帧抖动。
- 支持像素辅助网格，方便检查角色是否对齐到统一网格。
- 支持橡皮擦工具，可手动擦除杂色或多余像素。
- 支持按网格切分精灵表，默认 `4 × 4`，也可自定义行列数。
- 支持选择、复制、删除和重排帧。
- 支持多行最终帧序列，每行可独立组织动作帧。
- 支持每行循环播放动画预览，并可调整播放帧间隔。
- 支持导出透明背景 PNG 或指定背景色 PNG。

## 使用流程

1. 点击“选择图片”，导入本地像素精灵表。
2. 在“去背景色”区域点击画布背景取色。
3. 调整去背景容差，观察预计透明化像素数量和画布预览。
4. 如需检查杂色，开启背景预览并选择一个对比明显的预览色。
5. 预览效果满意后，点击“应用为透明”写入透明像素。
6. 如果角色轮廓残留红边、蓝边等背景色边，在“像素清理与对齐”区域执行“去背景色边”。
7. 再执行“自动去毛刺”，必要时调整 Alpha 阈值、邻居下限和小色块阈值。
8. 如动作帧位置抖动，按当前切分行列执行“按帧底部居中对齐”。
9. 如需要检查统一网格，开启“显示网格”并调整网格间距。
10. 如仍有杂色，开启橡皮擦并拖动画布手动擦除。
11. 设置切分行列数，生成独立帧。
12. 选择需要的帧，添加到当前最终序列行。
13. 新增多行序列，为不同动作或方向组织帧。
14. 使用每行播放预览检查动作效果。
15. 选择透明背景或指定背景色后导出多行 PNG。

## 核心算法原理

### 背景色去除

背景色去除基于用户取到的背景色和容差阈值进行 RGB 距离判断。

1. 用户在画布上点击背景像素，系统记录该像素的 RGB 颜色。
2. 遍历当前图片中的所有非透明像素。
3. 计算每个像素与目标背景色的颜色距离。
4. 如果距离小于等于当前容差，则将该像素的 Alpha 设置为 `0`，也就是变为透明。

这种方式适合纯色或近似纯色背景。AI 生成图片常常不是完全纯色，所以容差不能过低；但容差过高也可能误删角色内部相近颜色。

### 去背景色边

AI 图片扣掉红底、蓝底或绿底后，角色边缘常会残留一圈背景色。这通常来自抗锯齿、颜色混合或半透明过渡像素。

工具中的“去背景色边”只处理透明区域附近的非透明像素，避免影响角色主体内部颜色。大致逻辑是：

1. 先找到靠近透明像素的边缘像素。
2. 计算该边缘像素和已取背景色之间的 RGB 距离。
3. 如果距离在边缘容差以内，认为它仍然是背景残留，直接设为透明。
4. 如果像素不是纯背景残留，但颜色仍然明显偏向背景色，则从附近更稳定的角色像素采样平均色。
5. 用采样得到的角色颜色重染该边缘像素，减轻红边、蓝边或绿边。

因此它包含两类处理：一类是“透明化背景色边”，另一类是“用角色邻近颜色修正混色边缘”。

### 自动去毛刺

自动去毛刺用于处理 AI 像素图中孤立、不规则的小噪点。

它主要结合三种判断：

1. **Alpha 阈值**：低于阈值的半透明像素被认为不稳定，可以清除。
2. **邻居数量**：检查 8 邻域内有多少相邻非透明像素，邻居过少的像素更可能是孤立噪点。
3. **连通块大小**：把相连的像素看成一个小区域，如果区域面积低于阈值，则认为是小杂色块。

这个算法适合清理孤立毛刺，但不适合用过高阈值一次性处理，否则可能误删角色眼睛、高光或细线等像素细节。

### 像素网格对齐

AI 生成的“伪像素”经常出现像素块大小不一致、边缘带渐变、局部没有对齐统一网格的问题。像素网格对齐的目标是把这些不稳定色块尽量整理成真正像素风的 `N × N` 纯色方块。

算法流程：

1. 将图片按指定像素块大小切成多个 `N × N` 小块。
2. 对每个小块统计非透明像素的颜色出现次数。
3. 找到小块中出现最多的颜色，作为该块的主色。
4. 如果小块内透明像素占多数，则整个块转为透明。
5. 否则用主色填满整个 `N × N` 小块，并把有效像素 Alpha 统一为不透明。

这样可以减少抗锯齿、渐变和不规则色块，让画面更接近“每个像素都是固定大小纯色方块”的真像素风。

工具也提供自动检测块大小：它会尝试多个候选块大小，采样判断每种块大小下的颜色一致性，选择得分更高的块大小作为建议值。

### 色彩量化

AI 输出的像素角色可能包含几百到几千种相近颜色，而传统像素角色通常使用较少颜色，例如 `16` 色或 `32` 色。色彩量化用于把大量相近颜色压缩到指定数量的调色板中。

本工具使用的是 Median Cut 思路：

1. 统计所有非透明像素的 RGB 颜色和出现次数。
2. 把所有颜色放进一个 RGB 颜色空间盒子中。
3. 找到颜色跨度最大的盒子，并沿跨度最大的通道（R、G 或 B）排序。
4. 按像素数量的加权中位数把盒子切成两半。
5. 重复切分，直到盒子数量达到目标颜色数。
6. 对每个盒子计算加权平均色，生成最终调色板。
7. 将图片中的每个非透明像素映射到距离最近的调色板颜色。

这种方式可以把“几十种近似红色”“几十种近似阴影色”合并为更少、更稳定的颜色。它不等同于艺术级手工调色板，但很适合快速降低 AI 像素图的颜色噪声。

### 预览与确认

像素网格对齐和色彩量化都属于破坏性处理：一旦直接应用，原始细节可能被合并或清除。因此工具采用“先预览，后确认”的方式：

1. 点击“预览网格对齐”或“预览色彩量化”时，只临时更新画布显示。
2. 如果效果满意，点击“确认应用预览”才写入当前图片数据。
3. 如果效果不对，点击“取消预览”，图片保持原状态。
4. 用户可以反复调整块大小、透明阈值或目标颜色数，多次预览后再确认。

这可以降低误操作成本，尤其适合处理 AI 输出质量差异较大的素材。

### 按帧底部居中对齐

动作帧抖动通常来自每一帧角色主体位置不一致。底部居中对齐会基于当前切分行列逐帧处理：

1. 对每个切分帧查找非透明像素的包围盒。
2. 计算角色主体宽高。
3. 将主体水平放到帧中心。
4. 将主体底部贴到帧底部。
5. 把移动后的像素写回对应帧区域。

这个算法适合人物站立、行走等底部接地的 RPG 角色帧，可以减少播放动画时的上下左右抖动。

## 注意事项

- 去背景操作分为预览和应用两步：取色、调容差只影响预览，点击“应用为透明”后才会修改当前图片数据。
- 去背景色边依赖当前取到的背景色，只处理透明区域附近的角色边缘像素。
- 自动去毛刺会直接修改当前图片数据，建议先用较小阈值尝试，避免误删角色细节。
- 按帧底部居中对齐会使用当前切分行列数，因此请先确认切分配置是否正确。
- 橡皮擦会直接修改当前图片数据，后续切分、重组和导出都会使用擦除后的结果。
- 导出多行 PNG 时，行顺序按界面中的最终序列行顺序排列，行内帧顺序从左到右排列。
- `node_modules`、`dist` 和本地环境文件不会提交到 Git。
