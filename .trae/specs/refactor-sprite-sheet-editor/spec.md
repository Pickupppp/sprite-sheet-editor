# 像素精灵表编辑器重构 Spec

## Why
当前项目核心功能已经基本完成，但主要业务、状态、图像算法与 UI 都集中在 `src/main.tsx` 中，单文件超过 3000 行，后续维护、排查问题和扩展功能的成本较高。需要在不改变现有用户体验和功能行为的前提下，对项目进行模块化重构，降低单文件复杂度并清理冗余写法。

## What Changes
- 将图像处理算法从页面组件中拆分到独立工具模块，便于复用和测试。
- 将画布渲染、文件导入、背景处理、像素清理、像素规范化、切帧、序列重组、动画预览、导出等 UI 区域拆分为更小的组件。
- 将通用类型、常量和纯函数移动到明确命名的模块中。
- 梳理 React 状态与事件处理逻辑，减少 `main.tsx` 的职责，使其主要负责应用装配。
- 清理明显重复或不规范的实现，保留现有功能、交互文案和导出结果。
- 重构后通过 TypeScript 诊断与项目构建验证。

## Impact
- Affected specs: 像素精灵表编辑器整体实现、画布编辑、背景处理、像素清理、像素规范化、帧切分、帧重组、动画预览、导出流程
- Affected code: `src/main.tsx`、`src/styles.css`、新增的 `src/components/`、`src/lib/`、`src/types/` 或同等模块目录

## ADDED Requirements
### Requirement: 模块化代码结构
系统 SHALL 将核心功能拆分到语义明确的模块中，避免单个入口文件承载全部业务逻辑。

#### Scenario: 入口文件职责收敛
- **WHEN** 开发者打开 `src/main.tsx`
- **THEN** 该文件主要负责 React 挂载和应用组件装配
- **AND** 不再包含大量图像算法、长篇 JSX 控制面板或重复工具函数

#### Scenario: 图像算法独立维护
- **WHEN** 开发者需要修改背景去除、去毛刺、网格对齐或色彩量化算法
- **THEN** 可以在独立的图像处理模块中定位相关纯函数
- **AND** 不需要在大型页面组件中搜索算法实现

### Requirement: 功能行为保持一致
系统 SHALL 在重构后保持现有用户可见功能与交互行为不变。

#### Scenario: 核心流程保持可用
- **WHEN** 用户导入图片、去背景、清理像素、规范化像素、切帧、重组并导出
- **THEN** 系统行为与重构前一致
- **AND** 不因模块拆分丢失已有功能

### Requirement: 可验证的重构结果
系统 SHALL 在重构完成后提供可运行、可构建的项目状态。

#### Scenario: 构建验证
- **WHEN** 开发者执行项目构建命令
- **THEN** TypeScript 编译和 Vite 构建成功
- **AND** 不出现由重构引入的诊断错误

## MODIFIED Requirements
### Requirement: 项目维护性
系统 SHALL 通过模块拆分、命名整理和冗余清理，提高代码可读性和维护性，同时保持现有功能规格不变。

## REMOVED Requirements
无。
